# 말랑말랑 한글 친구

유치원생이 단어 또는 짧은 문장을 말하면, 서버형 AI가 한국어로 전사하고 더 자연스러운 한국어 음성으로 다시 읽어주는 웹앱입니다.

## 포함된 기능

- `OpenAI gpt-4o-transcribe` 기반 한국어 음성인식
- `OpenAI gpt-4o-mini-tts` 기반 자연스러운 한국어 읽어주기
- `단어 놀이` / `문장 놀이` 모드 전환
- 인식 완료 후 한글 결과 문장 표시
- 옆의 소리 버튼으로 AI 목소리 재생
- 최근에 말한 내용이 위로 쌓이는 `아이 말 기록장`
- 유치원 느낌의 아기자기한 SVG 캐릭터 이미지 2종
- 모바일도 고려한 반응형 화면

## 실행 방법

### 1. 환경 변수 준비

`.env.example`을 참고해서 프로젝트 루트에 `.env` 파일을 만들면 AI 음성인식과 더 자연스러운 AI 읽어주기를 사용할 수 있습니다.

```env
OPENAI_API_KEY=sk-...
PORT=4173
OPENAI_STT_MODEL=gpt-4o-transcribe
OPENAI_TTS_MODEL=gpt-4o-mini-tts
OPENAI_TTS_VOICE=sage
```

### 2. 서버 실행

```bash
cd "/Users/lee/Documents/Korean Recognition"
npm start
```

### 3. 브라우저 열기

브라우저에서 `http://localhost:4173`을 엽니다.

추천 브라우저:

- 데스크톱 Chrome
- 데스크톱 Edge
- Android Chrome

브라우저는 마이크 녹음을 담당하고, 실제 인식과 읽어주기는 서버가 OpenAI Audio API로 처리합니다.

`.env`가 아직 없더라도 앱은 브라우저 기본 음성인식/읽어주기 폴백으로 테스트할 수 있습니다. 다만 이 경우 목소리와 인식 품질은 더 제한적입니다.

## 현재 구조

1. 아이가 큰 마이크 버튼을 누릅니다.
2. 브라우저가 목소리를 녹음합니다.
3. 녹음한 음성을 서버가 OpenAI 전사 API로 보냅니다.
4. 전사된 한글 문장을 화면에 보여줍니다.
5. 전사된 문장은 `아이 말 기록장`에 저장되고 가장 최근 기록이 맨 위로 옵니다.
6. 소리 버튼을 누르면 서버가 OpenAI TTS로 MP3를 만들어 재생합니다.

## 지금과 예전 방식의 차이

### 예전 브라우저 기본 방식

- 브라우저 안의 기본 음성인식과 기본 읽어주기에 의존합니다.
- 구현은 빠르지만 브라우저와 운영체제에 따라 품질 차이가 큽니다.
- 목소리가 기계적으로 들릴 수 있고, 유아 발화 보정이 제한적입니다.

### 지금 AI 서버 방식

- 녹음한 소리를 서버에서 AI 음성인식으로 처리합니다.
- 읽어주기도 AI 음성으로 생성합니다.
- 한국어 발화 품질과 음성 자연스러움을 더 잘 제어할 수 있습니다.
- 대신 API 키, 네트워크, 비용 관리가 필요합니다.

## 실제 앱에서 권장하는 진행 순서

### 1. 문제 정의와 교실 시나리오 정리

- 아이가 `한 단어`를 말하는 상황과 `한 문장`을 말하는 상황을 나눠 정리합니다.
- 교실 소음, 마이크 거리, 말 빠르기, 어눌한 발화를 실제 사용 환경 기준으로 적어둡니다.

### 2. 유치원 단어 사전 만들기

- 색깔, 동물, 과일, 인사말, 교실 사물처럼 자주 쓰는 단어를 먼저 모읍니다.
- 문장 패턴도 함께 정리합니다.
- 예: `저는 토끼가 좋아요`
- 예: `오늘은 비가 와요`

### 3. 아이 발화 테스트 데이터 모으기

- 보호자 및 기관 동의 절차를 거친 뒤 샘플 발화를 수집합니다.
- 정확히 틀리는 발음, 자주 섞이는 단어, 교실 소음 환경을 같이 기록합니다.

### 4. 웹 프로토타입 다듬기

- 지금 만든 화면에서 버튼 크기, 문장 글자 크기, 다시 듣기 흐름을 검증합니다.
- 선생님이 아이와 함께 쓰기 쉬운지 현장에서 확인합니다.

### 5. 단어 사전과 후처리 규칙 강화

- 자주 틀리는 단어를 교실 단어 사전에 추가합니다.
- 발화가 어눌해도 의도를 유지하도록 후처리 규칙을 붙입니다.

### 6. 현장 파일럿

- 실제 수업에서 써 보며 어떤 말이 잘 안 들리는지 확인합니다.
- 재시도 버튼과 안내 문구를 더 쉽게 다듬습니다.

### 7. 운영 기능 추가

- 오인식 로그 수집
- 학급별 자주 쓰는 단어 세트
- 기기별 마이크 설정 가이드
- 네트워크 문제 시 재시도 처리

## GitHub Pages로 무료 배포하는 방법

아래는 비용 없이 공개 링크를 만드는 `GitHub Pages` 기준 배포 순서입니다.

중요:

- 이 방식은 `무료 모드`입니다.
- 즉, `브라우저 기본 음성인식 + 브라우저 기본 읽어주기 + 말 기록장`으로 동작합니다.
- `AI 음성인식`과 `AI 자연 음성 읽어주기`는 GitHub Pages만으로는 사용할 수 없습니다.

### 1. GitHub 저장소 만들기

- GitHub에서 새 저장소를 만듭니다.
- 현재 프로젝트 파일 전체를 그 저장소에 올립니다.

### 2. 로컬에서 화면 확인

- 프로젝트 루트에서 `npm start`를 실행합니다.
- 브라우저에서 `http://localhost:4173`을 열어 화면이 보이는지 확인합니다.
- `.env`가 없어도 기본 모드로 동작합니다.

### 3. GitHub에 코드 올리기

- 로컬에서 Git 저장소가 아니라면 아래처럼 시작합니다.

```bash
cd "/Users/lee/Documents/Korean Recognition"
git init
git add .
git commit -m "Initial commit"
```

- GitHub 저장소를 remote로 연결합니다.

```bash
git remote add origin https://github.com/사용자이름/저장소이름.git
git branch -M main
git push -u origin main
```

### 4. GitHub Pages 켜기

- GitHub 저장소 화면으로 갑니다.
- `Settings` 탭을 엽니다.
- 왼쪽 메뉴에서 `Pages`를 누릅니다.
- `Source`에서 `Deploy from a branch`를 선택합니다.
- Branch는 `main`
- Folder는 `/(root)`를 선택합니다.
- `Save`를 누릅니다.

GitHub 공식 문서도 이 방식인 `branch + root` 배포를 권장합니다.

### 5. 배포 완료 기다리기

- 잠시 기다리면 GitHub가 사이트를 배포합니다.
- 보통 주소는 아래 형태입니다.

```text
https://사용자이름.github.io/저장소이름/
```

### 6. 배포 후 첫 확인

- 배포 URL을 엽니다.
- 마이크 권한을 허용합니다.
- 단어 놀이와 문장 놀이를 각각 한 번씩 테스트합니다.
- 읽어주기 버튼과 `아이 말 기록장`이 잘 동작하는지 확인합니다.

### 6-1. 휴대폰에서 쓰는 방법

- GitHub Pages 주소는 `https://`로 열리므로 휴대폰에서도 마이크 권한 요청은 가능합니다.
- 무료 모드는 `브라우저 기본 음성인식`을 쓰기 때문에 `안드로이드 Chrome/Edge` 쪽이 가장 안정적입니다.
- `아이폰/iPad`에서는 브라우저 기본 음성인식 지원이 제한될 수 있어 같은 링크라도 인식 버튼이 기대만큼 동작하지 않을 수 있습니다.
- 아이폰까지 안정적으로 쓰고 싶으면 나중에 `Render` 같은 서버형 배포로 옮겨 `AI 음성인식` 모드를 사용하는 편이 좋습니다.
- 홈 화면에 추가해 두면 앱처럼 바로 열 수 있습니다.

### 7. 다른 사람에게 공유

- GitHub Pages 주소를 그대로 공유하면 됩니다.
- 유치원 선생님이나 보호자가 링크를 눌러 바로 사용할 수 있습니다.

### 8. 나중에 주소를 예쁘게 바꾸기

- 자체 도메인이 있으면 GitHub Pages의 Custom Domain 기능으로 연결합니다.
- 예: `speech.yourkindergarten.kr`

### 9. 운영 전 꼭 확인할 것

- 마이크 권한 안내 문구가 이해하기 쉬운지
- 아이 말 기록이 브라우저에만 저장되는지
- 다른 기기와 기록이 자동으로 공유되지 않는다는 점을 이해했는지
- 교실 소음 환경에서 인식률이 괜찮은지

## 나중에 AI 목소리까지 쓰고 싶다면

- GitHub Pages는 정적 사이트 호스팅이라 비밀 API 키를 안전하게 둘 서버가 없습니다.
- 그래서 더 자연스러운 AI 읽어주기와 AI 음성인식까지 쓰려면 나중에 `Render`, `Railway`, `Vercel Functions` 같은 서버형 배포로 옮겨야 합니다.
